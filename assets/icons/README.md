# morgward icons

Source of truth: **`icon.svg`** (terminal-shield, `>_` prompt; shield `#1b222c→#0d1117`,
accent `#39d353`). Everything else is generated from it.

## Files

- `icon.svg` — master vector. Edit this, then regenerate the rest.
- `icon-{16,32,48,64,128,256,512}.png` — transparent PNG raster set.
- `favicon.ico` — multi-res Windows/favicon (16/32/48/256, PNG-compressed).
- `build-png.ps1` — SVG → PNG set (headless Chrome/Edge, crops to exact square).
- `build-ico.ps1` — PNG set → `favicon.ico`.

## Regenerate (Windows dev host)

```powershell
.\assets\icons\build-png.ps1   # icon.svg -> icon-*.png
.\assets\icons\build-ico.ps1   # icon-*.png -> favicon.ico
```

## Where the icon actually shows (per OS)

A single CLI/TUI binary cannot carry one OS-shown icon everywhere — a terminal draws
its own icon while morgward runs. The app icon appears in file managers / launchers /
taskbars only, and the mechanism differs:

- **Windows** — embedded **inside** the `.exe` via `cmd/morgward/rsrc_windows_*.syso`
  (generated by `make winres` from `cmd/morgward/winres/winres.json`; committed, so
  `go build` links it automatically). Shows in Explorer / taskbar.
- **Linux** — sidecar `.desktop` + hicolor PNGs under `packaging/linux/`, shipped as the
  `dist/desktop/morgward-linux-*-desktop.tar.gz` release tarball (`make package-linux`
  or `build.ps1`). `install.sh` drops them into the XDG dirs → app menu.
- **macOS** — not wired (would require a `.app` bundle); intentionally out of scope.
